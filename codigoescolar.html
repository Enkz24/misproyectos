Sistema Escolar de Asistencia con QR | Arquitectura + Código Base

Propuesta visual y técnica de un sistema escolar donde cada estudiante registra su asistencia mediante un código QR único. La solución considera frontend desplegado en Vercel, autenticación y API con Supabase, repositorio del código en GitHub y persistencia en PostgreSQL para cursos, alumnos, horarios y registros de entrada.

Objetivo del sistema

Digitalizar el pase de lista para reducir errores manuales, acelerar el registro en aula y permitir trazabilidad en tiempo real. Cada alumno cuenta con un QR asociado a su matrícula, y el docente o personal autorizado puede escanearlo desde una interfaz web responsiva.

Stack sugerido: Next.js para la app, Supabase para auth y servicios backend, PostgreSQL como base de datos principal, Vercel para despliegue continuo y GitHub para versionado, issues y control del código fuente.

Componentes principales

  • Panel de administración para alumnos, docentes, grados y secciones.
  • Generación de QR por estudiante con vínculo a matrícula y estado activo.
  • Registro de asistencia por horario, curso, fecha y tipo de marcación.
  • Reportes por aula, estudiante y rango de fechas.
  • Historial auditable alojado en PostgreSQL mediante Supabase.

Modelo de datos en PostgreSQL

Esquema base para Supabase SQL / PostgreSQL 
create table students (
  id uuid primary key default gen_random_uuid(),
  code varchar(20) unique not null,
  full_name text not null,
  grade text not null,
  section text not null,
  qr_token text unique not null,
  is_active boolean default true,
  created_at timestamptz default now()
);

create table courses (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  teacher_name text not null,
  weekday int not null,
  start_time time not null,
  end_time time not null,
  created_at timestamptz default now()
);

create table attendance (
  id uuid primary key default gen_random_uuid(),
  student_id uuid not null references students(id) on delete cascade,
  course_id uuid not null references courses(id) on delete cascade,
  attendance_date date not null,
  check_in_time timestamptz default now(),
  status varchar(20) not null default 'presente',
  registered_by text,
  created_at timestamptz default now()
);

create unique index uq_attendance_once
on attendance(student_id, course_id, attendance_date);

Qué resuelve esta estructura

La base separa estudiantes, cursos y asistencias para facilitar consultas, reportes y control de duplicados. El índice único evita que un alumno sea marcado dos veces en el mismo curso y fecha.

Registro de asistencia mediante escaneo QR

Endpoint para marcar asistencia Next.js + Supabase 
import { createClient } from '@supabase/supabase-js';

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
);

export async function registerAttendance(qrToken: string, courseId: string) {
  const today = new Date().toISOString().slice(0, 10);

  const { data: student, error: studentError } = await supabase
    .from('students')
    .select('id, full_name, is_active')
    .eq('qr_token', qrToken)
    .single();

  if (studentError || !student || !student.is_active) {
    throw new Error('Alumno no válido o inactivo');
  }

  const { data: existing } = await supabase
    .from('attendance')
    .select('id')
    .eq('student_id', student.id)
    .eq('course_id', courseId)
    .eq('attendance_date', today)
    .maybeSingle();

  if (existing) {
    return { ok: false, message: 'La asistencia ya fue registrada hoy' };
  }

  const { error: insertError } = await supabase
    .from('attendance')
    .insert({
      student_id: student.id,
      course_id: courseId,
      attendance_date: today,
      status: 'presente',
      registered_by: 'scanner-web'
    });

  if (insertError) throw insertError;

  return { ok: true, message: `Asistencia registrada para ${student.full_name}` };
}

Lógica funcional

Al escanear el QR, el sistema identifica al estudiante, valida que esté activo y comprueba si ya existe una asistencia para ese curso en la fecha actual. Si todo es correcto, guarda el registro.

Esta lógica puede ejecutarse desde una API Route o Server Action en Next.js, manteniendo segura la llave de servicio de Supabase en variables de entorno desplegadas en Vercel.

Consulta de reportes y panel administrativo

Reporte diario por curso Supabase Query 
export async function getDailyAttendance(courseId: string, date: string) {
  const { data, error } = await supabase
    .from('attendance')
    .select(`
      id,
      attendance_date,
      status,
      check_in_time,
      students (
        full_name,
        code,
        grade,
        section
      )
    `)
    .eq('course_id', courseId)
    .eq('attendance_date', date)
    .order('check_in_time', { ascending: true });

  if (error) throw error;
  return data;
}

Utilidad del módulo

El panel administrativo permite revisar la asistencia del día, detectar tardanzas y exportar información para coordinación o secretaría. También se puede filtrar por grado, sección y docente.

Con Supabase es posible complementar esto con Row Level Security para que cada perfil vea solo la información que le corresponde dentro del sistema escolar.
